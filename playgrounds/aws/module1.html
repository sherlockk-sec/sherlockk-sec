Cloud Computing = On-demand delivery of IT resources over the internet with pay-as-you-go pricing

Traditional On-Premises

  • Buy physical servers
  • Maintain hardware
  • Fixed capacity
  • High upfront costs
  • You manage everything

  • Rent resources on-demand
  • No hardware maintenance
  • Scalable capacity
  • Pay-as-you-go
  • Provider manages infrastructure

üéØ Cloud Service Models (IaaS, PaaS, SaaS)

üñ•Ô∏è IaaS

Infrastructure as a Service

You manage: OS, Runtime, Apps

Provider manages: Virtualization, Servers, Storage, Networking

AWS Examples: EC2, S3, VPC

Use cases: Web hosting, Big data analytics

‚öôÔ∏è PaaS

Platform as a Service

You manage: Applications & Data

Provider manages: Runtime, OS, Infrastructure

AWS Examples: Elastic Beanstalk, Lambda

Use cases: App development, API management

üì± SaaS

Software as a Service

You manage: Just use the software!

Provider manages: Everything else

Examples: Gmail, Office 365, Salesforce

Use cases: Email, CRM, Collaboration tools

Remember the Email Example: IaaS = Rent server & install email software | PaaS = Use email platform, just configure | SaaS = Use Gmail directly

üéØ KEY EXAM POINTS TO REMEMBER:

  • 5 Essential Characteristics: On-demand, Broad access, Pooling, Elasticity, Measured
  • IaaS vs PaaS vs SaaS: Know the responsibility models
  • Region Selection: Compliance, Latency, Services, Pricing
  • AZ Naming: Region code + letter (e.g., us-east-1a)
  • Edge Locations: For CloudFront CDN, not for deploying EC2
  • Availability Zones: Physically separate data centers in a region
  • Public Cloud: AWS, Azure, GCP are examples
  • Hybrid Cloud: Combination of public and private clouds
